# 2026-04-10 오픈소스 사용 지침

## 오늘 문서 요약
- 이 프로젝트에서 오픈소스 패키지를 어떻게 선택하고 관리할지 기준을 정리했습니다.
- 무작정 새 패키지를 늘리지 않고, 유지보수가 가능한지 먼저 보도록 만들었습니다.
- 라이선스, 보안, 버전 관리, 업데이트 방식 중심으로 적었습니다.

## 이 문서의 목적
이 문서는 `animhandoff-api`에서 오픈소스를 사용할 때
"어떤 기준으로 추가할지", "어떻게 업데이트할지", "어떤 점을 조심할지"를 빠르게 확인하기 위한 문서입니다.

## 현재 사용 중인 주요 패키지
- `express`
- `cors`
- `multer`
- `fluent-ffmpeg`
- `ffmpeg-static`
- `@aws-sdk/client-s3`

## 가장 중요한 원칙
1. 패키지는 꼭 필요한 것만 추가합니다.
2. 비슷한 역할의 패키지를 여러 개 넣지 않습니다.
3. 설치 전에 유지보수 상태를 확인합니다.
4. 라이선스를 확인합니다.
5. 버전은 `package-lock.json`으로 고정해서 같이 관리합니다.

## 새 패키지를 추가할 때 확인할 것
새 패키지를 넣기 전에 아래를 먼저 봅니다.
- 마지막 업데이트 시점이 너무 오래되지 않았는지
- 다운로드 수가 너무 낮지 않은지
- 이슈가 방치된 상태가 아닌지
- 라이선스가 문제가 없는지
- 지금 코드로 직접 해결 가능한데 굳이 패키지를 늘리는 건 아닌지

## 특히 조심할 패키지
### `fluent-ffmpeg`
현재 설치는 되어 있지만, 이 패키지는 이미 오래되어 더 이상 활발히 관리되지 않습니다.
그래서 지금은 빠르게 서버를 세팅하기 위해 사용했지만,
장기적으로는 아래를 고려해야 합니다.
- 유지할지
- 직접 ffmpeg CLI 호출로 바꿀지
- 더 안정적인 대체 방법이 있는지

즉,
`fluent-ffmpeg`는 지금 당장 쓸 수는 있지만, 장기 운영 기준으로는 주기적으로 다시 판단해야 하는 패키지입니다.

## 버전 관리 원칙
- `package.json`만 보지 말고 `package-lock.json`도 같이 커밋합니다.
- 팀원이 설치해도 같은 버전이 깔리게 유지합니다.
- 큰 버전 업그레이드는 한 번에 여러 개 하지 않습니다.

## 업데이트 원칙
- 지금 잘 도는 서버는 이유 없이 전부 최신으로 올리지 않습니다.
- 보안 이슈가 있거나, 실제 문제가 있을 때 우선 검토합니다.
- 업데이트 후에는 최소한 아래를 다시 확인합니다.
  - 서버 실행
  - `/health`
  - 파일 업로드
  - FFmpeg 변환
  - R2 업로드

## 라이선스 관련 원칙
새 패키지를 추가할 때는 최소한 라이선스 이름을 확인합니다.
보통 아래처럼 생각하면 됩니다.
- MIT: 일반적으로 많이 사용
- Apache-2.0: 일반적으로 많이 사용
- GPL 계열: 사용 전에 더 신중히 검토

이 프로젝트는 외부에 공개될 수 있으므로,
"일단 깔고 보자"보다 "이 패키지를 써도 되는지"를 먼저 보는 편이 안전합니다.

## 오픈소스 코드 복사 원칙
인터넷 예제 코드를 가져올 때는 아래를 지킵니다.
- 출처를 확인합니다.
- 그대로 붙여넣기보다 현재 프로젝트 구조에 맞게 고칩니다.
- 이해하지 못한 코드는 넣지 않습니다.
- 민감한 값이 섞여 있지 않은지 확인합니다.

## 현재 프로젝트에 맞는 현실적인 기준
이 프로젝트는 초경량 API라서 패키지가 너무 많아지면 관리가 어려워집니다.
그래서 기준은 아래처럼 잡는 것이 좋습니다.
- 파일 업로드: `multer` 유지
- HTTP 서버: `express` 유지
- R2 업로드: `@aws-sdk/client-s3` 유지
- FFmpeg: 지금은 `fluent-ffmpeg + ffmpeg-static`
- 나머지는 실제 필요가 생길 때만 추가

## 패키지 추가 전 질문
새 패키지를 넣기 전에 아래 질문에 답할 수 있어야 합니다.
1. 이 패키지가 꼭 필요한가?
2. 기존 패키지로 대체 가능한가?
3. 유지보수 상태는 괜찮은가?
4. 라이선스는 안전한가?
5. 서버 용량과 시작 속도에 부담을 주지 않는가?

## 삭제 기준
아래에 해당하면 제거를 검토합니다.
- 실제 코드에서 더 이상 안 씀
- 비슷한 역할의 다른 패키지가 이미 있음
- 보안 이슈가 계속 남음
- 유지보수가 사실상 중단됨
