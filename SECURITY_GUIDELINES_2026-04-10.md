# 2026-04-10 보안 지침

## 오늘 문서 요약
- 이 프로젝트에서 꼭 지켜야 하는 기본 보안 기준을 정리했습니다.
- 지금 코드가 이미 하고 있는 것과, 배포 전에 꼭 확인해야 하는 것을 나눠 적었습니다.
- 운영 중 실수로 키가 노출되거나 `/tmp` 파일이 쌓이는 일을 막는 데 초점을 맞췄습니다.

## 이 문서의 목적
이 문서는 `animhandoff-api`를 운영할 때
"무엇을 절대 Git에 올리면 안 되는지", "어떤 값은 Render 환경 변수로만 관리해야 하는지", "파일 처리 후 무엇을 정리해야 하는지"를 빠르게 확인하기 위한 문서입니다.

## 가장 중요한 원칙
1. 비밀 값은 코드에 넣지 않습니다.
2. 업로드된 원본 파일과 변환 결과 파일은 `/tmp`에 오래 남기지 않습니다.
3. 외부 입력값은 그대로 믿지 않고 검증합니다.
4. 오픈소스 패키지는 버전과 출처를 확인한 뒤 씁니다.
5. 배포 후에는 실제 로그와 응답을 한 번 더 확인합니다.

## 절대 Git에 올리면 안 되는 것
아래 값은 절대 커밋하면 안 됩니다.
- `R2_ACCESS_KEY_ID`
- `R2_SECRET_ACCESS_KEY`
- `OBJECT_STORAGE_ACCESS_KEY_ID`
- `OBJECT_STORAGE_SECRET_ACCESS_KEY`
- Render에서 쓰는 실제 환경 변수 값 전체
- 테스트용 실사용 토큰
- 사용자 개인 파일

메모:
- `.env`는 이미 `.gitignore`에 들어 있습니다.
- 실제 값은 `.env.example`에 넣지 말고 예시만 적습니다.

## 환경 변수 관리 원칙
실제 배포 값은 아래 위치에서만 관리합니다.
- 로컬 테스트: `.env`
- 운영 배포: Render 환경 변수

코드에 하드코딩하면 안 되는 값 예시:
- R2 endpoint
- R2 access key
- R2 secret key
- bucket name
- public URL

## 업로드 파일 처리 원칙
이 프로젝트는 업로드 파일을 `/tmp`에 잠깐 저장합니다.
중요한 점은 아래입니다.
- 변환 성공 후 바로 삭제
- 업로드 실패해도 바로 삭제
- 예외가 나도 `finally`에서 삭제

이 규칙을 지키지 않으면 무료 서버에서 디스크가 금방 찹니다.

## 파일 검증 원칙
현재 서버는 업로드 파일을 받아서 바로 FFmpeg에 넘깁니다.
배포 후에는 아래 항목을 추가로 점검하는 것이 좋습니다.
- 허용할 확장자 또는 MIME 타입 제한
- 파일 최대 크기 제한
- 변환할 수 없는 손상 파일 처리

한 줄로 말하면,
"업로드됐다고 해서 바로 안전한 파일이라고 보면 안 됩니다."

## 로그 관리 원칙
서버 로그에는 아래 내용을 남겨도 됩니다.
- 변환 시작
- 변환 진행률
- 업로드 성공/실패
- 에러 메시지

하지만 아래는 로그에 남기면 안 됩니다.
- Access Key
- Secret Key
- 전체 Authorization 헤더
- 사용자 비밀 토큰

## CORS 관련 메모
현재 코드는 `cors()` 기본값을 쓰고 있어서 열려 있는 편입니다.
혼자 쓰는 단계에서는 빠르게 붙이는 데 도움이 되지만,
공개 서비스로 넓힐 때는 허용 origin을 제한하는 것이 더 안전합니다.

## 배포 전 체크리스트
- `.env`가 커밋되지 않았는지 확인
- Render 환경 변수 입력이 끝났는지 확인
- `/health`가 200으로 응답하는지 확인
- 샘플 파일 변환 후 `/tmp`에 파일이 남지 않는지 확인
- R2 업로드 후 URL이 실제로 열리는지 확인

## 운영 중 체크리스트
- 에러 로그가 반복되는지 확인
- 큰 파일 업로드가 계속 들어오는지 확인
- 변환 실패 후 `/tmp`가 남지 않는지 확인
- R2에 쓰레기 파일이 쌓이지 않는지 확인

## 사고가 났을 때 먼저 할 일
1. 환경 변수 노출 여부 확인
2. 노출된 키가 있으면 즉시 재발급
3. Render 로그 확인
4. R2 업로드 실패/성공 여부 확인
5. `/tmp` 정리 실패 여부 확인

## 지금 코드 기준으로 이미 반영된 것
- `/tmp` 정리 로직 있음
- `.gitignore` 있음
- R2 값은 환경 변수로 읽음
- 변환과 업로드 실패 시 에러 응답 반환

## 다음에 보강하면 좋은 것
- 허용 파일 형식 제한 강화
- CORS origin 제한
- 요청당 timeout 가드
- 업로드 파일 MIME 검증 강화
- 요청 로그에 request id 붙이기
